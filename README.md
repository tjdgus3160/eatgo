# 레스토랑 예약 시스템

## 도메인 모델링
+ Restaurant
+ Menu Item
+ Food & Beverages
+ User
+ Favorite
+ Review

## 시스템 아키텍처
Mobile Application

+ 3-tier Architecture
  + Front-end(Presentation - 사용자와 소통)=>HTML/CSS/JS
  + Back-end(Business - 사용자의 요청 처리)=>REST API
  + Database(Data Source - 데이터 저장)=>DBMS

+ Layered Architecture(Back-end)
  + UI Layer
  + Application Layer
  + Domain Layer
  + Infrastructure Layer

## 기술선택
- Java
- Spring Boot
- REST API
